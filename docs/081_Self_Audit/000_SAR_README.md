## 🎯 SAR 작성 가이드

### 언제 SAR을 작성하는가?

```
✅ SAR 작성 대상:
  ├─ Self Check 미통과 항목
  ├─ Self Test 테스트 실패
  ├─ 커버리지 부족 (80% 미달)
  ├─ 보안 검증 실패
  └─ ZEN_A4 자동 리뷰 오류

❌ SAR 불필요:
  ├─ 단순 오타 (커밋 메시지만 기록)
  └─ 경미한 스타일 이슈
```

### SAR 작성 절차

```
1. 오류 발견 (Self Check/Self Test)
2. 오류 분석 및 원인 파악
3. SAR 파일 생성 (SAR_YYYY-MM-DD_NNN_문제_분류.md)
4. SAR 작성 (현상, 원인, 조치, 검증, 예방)
5. 이 README에 목록 추가
6. Check List 업데이트 (동일 오류 재발 방지)
```

## 📌 자주 묻는 질문 (FAQ)

### Q1: SAR이 필수인가?

**A:** 네. Self Check/Self Test에서 발견된 모든 오류는 반드시 SAR로 기록해야 합니다.  
이는 다음을 보장합니다:
- 오류 추적성
- 동일 오류 재발 방지
- 팀 학습

### Q2: SAR 작성에 얼마나 시간이 걸리는가?

**A:** 평균 15-30분입니다.
- 오류 분석: 5-10분
- SAR 작성: 8-15분
- Check List 업데이트: 3-5분

### Q3: Check List 업데이트는 필수인가?

**A:** 네. SAR 작성의 핵심은 **다음 작업에서 동일 오류를 방지**하는 것입니다.  
Check List 업데이트 없이는 SAR의 가치가 반감됩니다.

### Q4: 이미 알려진 오류는 어떻게 하는가?

**A:** 기존 SAR을 참조하고, 새로운 SAR을 작성하지 않으며, Check List 항목이 이미 추가되어 있는지 확인합니다.  
필요시 기존 SAR의 개정 이력에 "재발 방지 확인"을 추가합니다.

### Q5: SAR을 언제까지 보관하는가?

**A:** 영구 보관합니다. SAR은 프로젝트의 학습 자산입니다.  
정기적으로 검토하여 유사 오류 패턴을 식별하고 예방합니다.
