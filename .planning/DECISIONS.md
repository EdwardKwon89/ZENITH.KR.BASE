### 3. Self Check & Self Test 절차 추가 (2026-04-08)

**컨텍스트:**
- 설계 및 구현 단계에서 품질 보증 강화 필요
- 자동 리뷰만으로는 부족한 부분 있음
- 개발자가 완료 전에 자체 검증해야 함

**결정:**

- Plan 단계 이후: **Self Check (설계 검증)** 추가
- Execute 단계 이후: **Self Test (자체 테스트)** 추가
- 조건: 둘 다 **통과해야만 다음 단계로 진행**

**근거:**

1. **설계 검증 (Self Check)**
   - 구현 전 설계가 완전한지 확인
   - 요구사항 누락 방지
   - 아키텍처 일관성 확보
   - 예상 이슈 조기 발견

2. **자체 테스트 (Self Test)**
   - 단위 테스트: 함수별 동작 검증
   - 커버리지: 80% 이상 보증
   - 수동 테스트: 사용자 관점 검증
   - 보안 검증: ZEN_A4 security-reviewer 통과

3. **실질적 효과**
   - 버그 조기 발견 (비용 $1 → $1,000 차이)
   - 리뷰 단계에서의 반려 감소
   - 코드 품질 일관성 유지
   - CI/CD 성공률 향상

**워크플로우 변경:**

```text
기존:
Design → Implement → Verify → Commit

개선:
Design → Self Check (필수) → ✅ 통과
   ↓
Implement → Self Test (필수) → ✅ 통과
   ↓
Verify → Claude Review → Commit
```

**자체 테스트 기준:**

```text
Phase 2 (Execute) 완료 조건:

├─ 모든 함수 구현 완료
├─ 테스트 코드 작성 완료
├─ npm test 모두 통과
├─ 커버리지 80% 이상
├─ 수동 기능 테스트 통과
├─ 보안 검증 통과
└─ Self Check 최종 확인 완료

조건 미충족 시:
  → Phase 3 진행 불가
  → Phase 2에서 재작업
  → 통과 후에만 진행
```

- 모든 기능 개발 프로세스
- 코드 완성 기준 명확화
